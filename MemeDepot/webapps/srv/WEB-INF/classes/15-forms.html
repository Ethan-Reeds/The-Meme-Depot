Forms
Motivation
  • Allow user to send information to website
  • Idea:
    • User enters data
    • Browser submits data to website
    • Server processes it
    • Output sent back to client
Forms
  • There are several ways to handle form data within web pages
  • We’ll cover exactly one way: Using Javascript to construct a submission
  • But first we must look at the HTML controls…
Input
  • Input = most flexible of form controls
  • Gives you a free-form text line:
    <input id=”city” size=20>
  • ID is optional, but it makes it easier to access the item later
Checkbox
  • Checkbox:
    <label><input type=’checkbox’ id=”sendSpamCheckbox” value=1 checked>Send me spam</label>
Radio Buttons
  • Radio buttons:
<div class='radioBox'>
    <label><input name='colorgroup' type='radio' value=1 checked>Red</label>
    <label><input name='colorgroup' type='radio' value=2>Green</label>
    <label><input name='colorgroup' type='radio' value=3>Blue</label>
</div>

<div class='radioBox'>
    <label><input name='patterngrp' type='radio' value=1 checked>Checked</label>
    <label><input name='patterngrp' type='radio' value=2>Plaid</label>
    <label><input name='patterngrp' type='radio' value=3>Paisley</label>
</div>
Text
  • For larger amounts of text
    <textarea id=’aboutMe’ rows=10 cols=40> Enter stuff here</textarea>
Select
  • Drop-down menus:
<select id=preferredOS size=1>
    <option selected name=penguin>Linux</option>
    <option name=osx>OS X</option>
    <option name=ios>iOS</option>
    <option name=beos>BeOS</option>
</select>
  • Listboxes are the same, but size is set to value > 1
File Upload
  • Upload file:
    <input id=”avatar” type=’file’>
Example
  • We’ll create a simple example form for our Jetty server
  • This assumes you have jQuery (as with our previous examples)
    • I.e., put jquery.min.js in webapps/ROOT
HTML
  • Create an HTML file webapps/ROOT/formtest.html
  • Create a Javascript file to actually do the form submission: webapps/ROOT/formtest.js
    • Note: Javascript files are available in Netbeans under “New File”
    • But: You have to go to “Other” in the type selection dialog box
  • Create a Java servlet to handle the server side processing. I called mine FormTest.java
    • Notice: We have a new @MultipartConfig option
    • And: We use doPost() instead of doGet()
  • Test it: Navigate to http://localhost:2020/formtest.html
Uploads
  • File uploads are a bit more work
  • Add a file input to the HTML: webapps/ROOT/formtest2.html
  • Change the Javascript: webapps/ROOT/formtest2.js
  • Create a new servlet FormTest2.java
  • Test it: Navigate to http://localhost:2020/formtest2.html
Validation
  • Maybe we want to do input validation (Ex: Make sure user doesn’t leave a required field blank)
  • First, create a CSS style for invalid input
  • Ex: Put this in a CSS file:
/* filename: foo.css */
.invalid { background: red; }
  • And then in the HTML file’s HEAD:
    <link rel=”stylesheet” href=”foo.css”>
Validation
  • Put this code at the top of submitForm():
let name = $("#name").val();
if( name.length === 0 ){
    $("#name").addClass("invalid");
    return;
} else {
    $("#name").removeClass("invalid");
}
